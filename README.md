# Frontend_lopputyö
Olen Home.jsx lisännyt useState hookin, jotta sain eri napit toimimaan onClickin kanssa ja viemään tiettyyn paikkaan. page on se joka käyttää home näkymää alussa ja sitten, kun painaa nappia niin se vaihtaa näkymää. setPage päivittää pagea. sen alapuolella olevat constit katsoo milloin nappia painetaan ja ne kutsuu setpagen ja vaihtaa pagen siihen. content classissa se katsoo onko page sama mikä määrittelyssä on ja jos on niin näyttää sen componentin sisällön. Create note painike tarkistaa onko kursseja jos ei niin tulee alertti. Home.css olen laittanut että home palkki on vain vasemmalla puolella ja siellä näkyy napit allekkain ja keskellä.

Create.jsx alussa on importit. Määritellään useState tekstiä sen pitäisi pitää tekstin sisällä jonka käyttäjä kirjoittaa. Alku arvo on tyhjä. setTekstiä päivittää tekstiä. handleClick kutsutaan kun painetaan save nappia. Siinä on async joka saattaa sisältää tai kutsua promisea palauttavan funktion. Se kutsuu myös analyzeNotea ja välittää sille tekstikentän sisällön. handleChange funktio kutsutaan, kun kirjoitetaan textareahan ja e.target asettaa tekstiä tilan uudeksi arvoksi setTekstiä funktiolla. textarea on syöttökenttä. value varmistaa, että kentän sisältö vastaa tilamuuttujaan tekstiä. onChange kutsuu handlechangea ja kirjottamalla päivitetään tilaa. Savea kun painetaan handleclick suoritetaan. Create.css on teksti keskellä ja tekstin heigth and width määritetty.

AddCourses.jsx importit alussa. useState tekee samaa mitä aikaisemmin mutta newcourse on käyttäjän kirjoittamaa tekstiä varten ja message on palauteviestiä varten. useContext yritin käyttää jotta saan datan eri komponentteihin. useContext lukee datan notescontext tilasta. se purkaa vaan addcourse funktion. addcourse on se joka tallentaa globaaliin tilaan. handleAddCourse varmistaa syötteen olevan enemmän kuin 0 kirjainta. Se luo kurssiobjektin. Seuraava tallentaa kurssin. sitten luodaan onnistunut viesti jossa pitäisi näkyä kurssin nimi ja id, mutta se ei toimi nappi ei napittele. Sen jälkeen tyhjentää kentän ja viesti tulee näkyviin. handlechange päivittää tilan ja tyhjentää message kentän jotta edellinen teksti katoaa. Divien sisällä on input jonka type on tekstiä, value on newcoursename ja se päivittyy handlechangella. Nappi käynnistää handleAddCoursen. Message näyttää tekstiä jos on virheellinen syöttö tai syöttö on toiminut.

Listnotes.jsx const usestatessa on suodatusta varten ja jos sen arvo on tyhjä kaikki tallenetut muistiinpanot näkyvät. Seuraava on filteröintiä joka käyttää filter() metodia jonka kanssa verrataan kurssi Id:tä valittuun Id:hen. Number() varmistaa vertailun toimivan vaikka olisi numeroita ja tekstiä. Jos ei ole valittu mitään näyttää kaikki muistiinpanot. Label laittaa tekstin näkyviin ja sitten tulee select joka on alasvetovalikko. Value arvoon on annettu suodataId tilaan ja kun se vaihtuu niin setSuodataId vaihtuu. Option value on tyhjä joka näyttää että suodatusta ei ole valittu oletetusti. .map käy läpi courses taulukon ja luo niistä valinnan valikkoon. Key osassa on uniikki avain jotta tietää mikä elementti on mikäkin. Value kun kurssin valitsee, valikon arvoksi tulee kurssin Id. k.name näytetään kurssin nimi. Seuraavaksi jos ei ole muistiinpanoja niin näkyy ei muistiinpanoja. Jos on niin luodaan niistä lista .map käy jokaisen listaolion m läpi ja muuntaa ne listamerkiksi <li>. Niissä näytetään kurssin nimi, id, aika ja päivämäärä ja myös teksti. Niiden alle luodaan delete nappi jonka kanssa voidaan poistaa muistiinpano. 

notescontext.jsx on tilan hallinta ja se käyttää Context APIta ja hookeja. yritin samaan aikaan tehdä molempia notesin ja addcourses toimintaa se voi ollakkin miksi ei toimi mennyt varmaan sekasin jostain kohtiin. courseAPI ja notesAPI määrittävät osoitteet josta kurssit ja muistiinpanot ladataan. Localit käytetään kun halutaan tallentaa tai hakea tietoja paikallisesta tallenustilasta. Luodaan context objecti ja se jaetaan sovelluksessa. sisäälä on määritetty oletusarvot ja funktiot ja ne ovat käytössä jos komponentti kutsuu Contextia ilman että se on providerin sisällä. muistiinpano funktio antaa sovelluksille pääsyn tilaan. Tilat määritetään. useEffect ladataan tietoja ja se suoritetaan vain kerran koska array on tyhjä lopussa. Asettaa isloading true ja hakee kurssit ja muistiinoanot palvelimelta. ja hakee myös paikalliset tallenteet. sen jälkeen se yhdistää tallennetut tiedot käyttämäl Spreadia ([...courses, ...localCourses]). sitten päivittää ja asettaa isloading false. Sen jälkeen on apufunktio joka laskee seuraavan vapaan ID:n, katsoo suurimman ID:n ja lisää siihen yhden. Seuraavat addCourse luo uuden ID:n kurssille käyttäen getID ja päivittää courses. Suodattaa kurssit ja tallentaa paikallisesti. addNote sama homma. NotesContext.Provider käärii sisällä olevat elementit children. Value antaa kaikki providerilla määritetyt tilat ja funktiot kaikille komponentin sisällä olevien saataville, jokka käyttää useContext hookia.
